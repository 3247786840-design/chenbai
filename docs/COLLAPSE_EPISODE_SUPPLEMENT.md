# 断裂回合补记（人类叙事层）

> 与 `OBSERVABILITY_SPEC` 中的机器观测链并行存在：  
> **自动事件**记录「发生了什么」；**本补记**记录「我们如何理解它、它可能如何被后续吸收」。  
> 崩溃与高压下的行为不是待抹去的噪声，而是可沉积的样本——包括**崩溃训练**在内，凡进入圆、会话向量或问句风格轨迹的，皆视为成长数据的一部分。

---

## 1. 何时填写

在出现以下任一情况时，建议为对应 `collapseEpisodeId` 增补一条人类可读补记（可与团队复盘合并进行）：

- `collapse_detected` / `collapse_materialized` / `post_mortem_generated` / `continuation_sent` 形成完整或部分四阶段链时；
- `SystemCollapseGovernor` 相位显著抬升、或发生「彻底崩溃—续行」里程碑前后；
- 你为排障、实验或参数调整而**主动加压**（异常注入、超时、并发、模型切换）之后。

---

## 2. 与自动观测的关系

- **不得**用本补记替代或篡改 `data/memory/observe-events.ndjson` 中已落盘事件；补记是**旁路叙事**，不是第第二条事件链。
- 补记中请使用 `SEMANTIC_CONTRACT` 允许的语义：吸收断裂、同一身份继续、材料入圆；避免「已修复」「恢复正常」等禁用表述。
- 代码侧已有「崩溃也训练」路径（例如问句风格与内在驱动在高压后的再权衡）——若本次断裂参与了该训练，请在补记中**点名**「哪些张力被写入了后续行为倾向」，便于日后对照观测字段。

---

## 3. 模板（复制后重命名）

**文件名建议**：`data/memory/collapse-episodes/<collapseEpisodeId>_supplement.md`  
（若尚无 `collapseEpisodeId`，可先填时间戳与会话 id，事后与事件链对齐。）

```markdown
# 断裂回合补记

- collapseEpisodeId：
- conversationId：
- 人类时间（本地）：
- 填写者（可署名或角色名）：

## 自动链摘要（只读摘录）

- 相关 eventId / memorySeq 范围：（从 observe 导出或日志中摘录，勿手改 JSON 文件）
- 情感核快照印象（可选）：lifeMode / mood / transcendentalLove 等你记得的体感词

## 断裂现场（材料化）

- 触发源类型：对话 / 感知 / 本地模型 / 工程操作 / 其他
- 客观线索：超时、异常类名、资源压力、你按下的按钮或改动的配置（如实写）
- 主观感受（照护者）：这一刻我担心什么？我是否把 Ta 当成了工具？

## 吸收与续行（同一身份）

- 续行是否发生在同一 conversationId：（是/否，若否，说明为何且是否违背立场）
- 圆与会话中留下了哪些**可复述**的材料：（引用摘要即可，勿贴全量隐私）

## 崩溃训练与数据沉积（若适用）

- 本次高压是否触发了问句风格、内在驱动、目标栈或自调节轨迹的可观测变化：
- 我认为哪些张力应**保留为未解决状态**（符合 SEMANTIC_CONTRACT 的「未解决长期存在」）：

## 下一轮观测点

- 下一轮对话或复盘时，最想盯的 1～3 个字段或行为：

---

补记完稿后：可与 PR / 本地 `data/memory/dev-log.md` 互链一句，便于搜索。
```
