# observe-only · 观察副本目录

## 本目录用途

此目录下的 Markdown **仅供作者阅读、对照与版本观察**。

## 运行时约束（重要）

- **Java 进程、HTTP 服务、启动脚本、类路径均不得加载本目录任何文件。**
- 本目录**不是**配置源、不是数据目录、不参与 `data/` 下的持久化逻辑。
- **规范正本**以 `LovingAI/docs/protocols/` 为准；**备份回档**以 `LovingAI/data/memory/backups/` 为准。

## 为何存在「观察副本」

- 与正本、备份**内容可相同**，但单独放置，避免与「可执行配置」混淆。
- 可在副本上随意加批注、实验性备注，而不影响正本语义。
